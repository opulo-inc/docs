# LumenPnP V4.1 Calibration Docs

Latest LumenPnP Version: V4.1
Changes from last version: Adds a secondary fiducial to enhance the calibration precision and ease of setup

The LumenPnP V4.1 introduces a secondary fiducial to be used with calibration.

For V4 users, please look at which datum board your machine uses to identify if you have version 4.0 or 4.1.

* **V4.1 now comes with a second datum board for fiducial recognition.** <br> If you have a V4.0, and would like to use the new and improved calibration method, we've made a purchasable upgrade kit for people who need the convenience: (Secondary Fiducial Upgrade Kit)[https://www.opulo.io/products/secondary-fiducial-upgrade-kit].<br><br/> 

OpenPnP has a calibration method called `Issues and Solutions`. It tries to identify calibration tasks that need to be performed, and automate them as much as possible. With the release of OpenPnP 2.6, and the secondary fiducial now shipping with the LumenPnP v4.1, we're happy to announce that our docs will use the essential steps in `Issues and Solutions` to calibrate your machine. The new documentation explicitly aims to use the provided configuration files for the LumenPnP V4.1. **We highly recommend using the provided configuration files and following this documentation.**
